Connectors — one home for tools, services, MCP & skills

A single page (replacing the separate Services, MCP and Workspace pages) where users connect integrations with their own account, install skills, choose Global vs a specific agent, and set allow / ask / deny on every tool. Design preview — not final pixels.

2Per-tool permissions
Allow · Ask · Deny — per tool
Read-only tools default to Allow; write/delete default to Ask. Set any tool to Deny to block it for this scope.
Write / delete tools 3
SSH_EXEC run remote commandAllowAskDeny
Delete a SequenceAllowAskDeny

🟢 Allow = runs automatically · 🟠 Ask = a human approval card appears before it runs · 🔴 Deny = blocked (added to the agent’s forbidden list).
② Permissions — reuses your existing risk tiers + approval gate; no new safety engine.
